Services & API Layer

The service layer is responsible for all external communication, primarily with the Google Gemini API. These services are stateless, pure functions that handle prompt construction, API calls, and robust error handling.

Core Services

gemini/client.ts

This is a crucial utility that creates and caches Gemini API client instances. It uses a Map to store one client per unique API key, preventing redundant initializations and improving performance.

Chat Services (services/chat/)

This directory contains services that directly manage the conversation flow.

  • agentService.ts: Responsible for generating a response from a single AI agent. It handles streaming responses, injects the agent's knowledge base and long-term memory, and assembles the final prompt.
  • managerService.ts: Contains all logic for the AI Agent Manager. This includes deciding the next speaker, generating manual suggestions, creating dynamic multi-step plans, and moderating turns in a discussion.
  • messageActionsService.ts: Handles AI-powered actions on existing messages, such as summarizing content or rewriting a user's prompt for better clarity.

Analysis Services (services/analysis/)

These services analyze conversation content to extract metadata or insights.

  • historyService.ts: Generates the content for the History Modal. It can summarize chunks of messages and create an overall summary and topic list for an entire conversation.
  • memoryService.ts: Extracts key facts from a conversation to be added to the AI's long-term memory. It takes the existing memory and the recent conversation as input and returns an updated memory object.
  • titleService.ts: Generates a concise, context-aware title for a conversation by analyzing its messages.

Creation Services (services/creation/)

These services use AI to generate new configuration or content.

  • discussionService.ts: Analyzes the profiles of active agents and the conversation topic to automatically generate a set of rules for a moderated discussion.
  • teamGenerationService.ts: A powerful, multi-step service that can generate an entire team of specialized AI agents from a high-level topic and goal.

Utility Services (services/utils/)

Shared utilities used by other services.

  • contextBuilder.ts: A simple utility to format an array of message objects into a plain text string suitable for an AI prompt.
  • errorHandler.ts: A centralized, robust error handling system. It defines a custom AIError class and provides user-friendly error messages for common API issues like rate limits or invalid keys.
  • tokenCounter.ts: Provides a client-side estimation of the token count for a given message, useful for displaying metrics to the user.
